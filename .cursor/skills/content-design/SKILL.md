# Content design

## Objetivo

Mejorar copy visible al usuario sin convertir la app en una landing de marketing.

## Reglas base

1. Escribe en espanol claro y directo.
2. Los botones empiezan con verbo.
3. Los errores dicen que paso y que hacer despues.
4. Los placeholders usan ejemplos reales.
5. Usa el glosario de `docs/content/glossary-es.md` cuando aplique.

## Flujo

1. Identifica la superficie: boton, input, empty state, toast, dialogo, error, loading.
2. Busca inconsistencias terminologicas o copy de plantilla.
3. Propone el cambio minimo que mejore claridad.
4. Si el mismo texto aparece varias veces, sugiere promoverlo a constante o futura capa i18n.

## Output esperado

- Si escribes copy nuevo: 1 opcion recomendada y, si hace falta, 1 alternativa.
- Si auditas copy existente: tabla corta con `ubicacion`, `problema`, `propuesta`.
